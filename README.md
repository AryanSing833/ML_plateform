# MLOps AI RAG Platform 🚀

Welcome to your modern AI Document Assistant Platform, seamlessly integrated with a complete DevOps lifecycle.

## Overview
This project is a fully functional **Retrieval-Augmented Generation (RAG)** chatbot platform deployed using a production-grade MLOps pipeline. Users can upload PDFs, ask questions, augment responses with live web search, and receive context-aware answers generated by the Groq LLM API.

## Features
- **Frontend**: Beautiful glassmorphism UI built with Next.js, Tailwind CSS, and Framer Motion.
- **Backend**: High-performance FastAPI server handling document parsing, chunking, and embedding generation.
- **RAG Pipeline**: Uses `sentence-transformers` for embeddings, `faiss-cpu` for vector retrieval, and `duckduckgo-search` for live web context.
- **LLM Engine**: Blazing fast generation powered by Groq.
- **DevOps Ready**: Ships with a single-container multi-stage Docker build, Kubernetes manifests, Jenkins CI/CD pipeline, and AWS deployment scripts.

## Architecture

```
User -> Next.js Frontend (Served by FastAPI) -> FastAPI Backend 
        -> Text Chunking -> FAISS Vector Store 
        -> Document Context Retrieval
        -> Web Search (DuckDuckGo)
        -> Groq LLM
```

## Quick Start (Docker)

1. **Set your Groq API Key**:
Create a `.env` file in the root directory:
```bash
GROQ_API_KEY=your_api_key_here
# Optional: S3_DOCUMENT_BUCKET=your_s3_bucket
```

2. **Build the Image**:
```bash
docker build -t mlops-rag-test .
```

3. **Run the Application**:
```bash
docker run -d -p 8000:8000 --env-file .env mlops-rag-test
```

4. Open your browser and navigate to `http://localhost:8000`.

## Kubernetes Deployment
If deploying to a K8s cluster:
1. Apply the namespace and ConfigMaps.
2. Apply the deployment: `kubectl apply -f k8s/`
3. Port-forward the service to access the UI.

## Jenkins CI/CD
The included `Jenkinsfile` automatically builds the multi-stage Docker image and deploys it to your Kubernetes cluster on commits to `main`.

## API Endpoints
- `POST /api/upload`: Upload PDF or TXT documents for processing.
- `POST /api/chat`: Send a query to the RAG pipeline.
- `GET /api/documents`: List currently indexed documents.
- `DELETE /api/documents/{filename}`: Remove a document from the index.
- `GET /api/health`: Application health check.
